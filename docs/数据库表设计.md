# 第四部分：数据库表设计

## 文档说明

本部分基于数据库设计V5.1.0（修正版），完整支持以下核心算法：
- 试错分配算法（递归回溯）
- 班次均衡调整（波浪交替）
- 约束校验算法
- 顺位排序算法

**数据表分类**：
- **T_CX_** 开头：成型排程业务表
- **T_LH_** 开头：硫化排程业务表
- **T_MDM_** 开头：主数据管理表

## 四、表关系图

### 4.1 核心业务关系

```
T_CX_SCHEDULE_RESULT (成型排程结果)
    ├── 1:N → T_CX_SCHEDULE_DETAIL (排程明细)
    │
    ├── N:1 → T_MDM_MOLDING_MACHINE (成型机台)
    ├── N:1 → T_CX_MATERIAL (胎胚物料)
    │
    └── 1:N → T_LH_SCHEDULE_RESULT (硫化排程结果)
            └── N:1 → T_MDM_MOLDING_MACHINE (硫化机台)

T_CX_MATERIAL (胎胚物料)
    └── 1:1 → T_CX_STOCK (胎胚库存)

T_CX_DAILY_EMBRYO_TASK (日胎胚任务)
    └── N:1 → T_CX_SCHEDULE_RESULT (排程结果)

T_CX_TRIAL_TASK (试制任务)
    └── N:1 → T_CX_MATERIAL (胎胚物料)

T_CX_STRUCTURE_ENDING (结构收尾管理)
    └── N:1 → T_MDM_STRUCTURE_NAME (结构名称字典)
```

### 4.2 配置关系

```
T_CX_PARAM_CONFIG (参数配置)
    ├── 支持排程算法参数
    └── 支持业务规则参数

T_CX_SHIFT_CONFIG (班次配置)
    └── 定义8个班次的时间信息

T_CX_ALERT_CONFIG (预警配置)
    └── 定义库存预警阈值

T_CX_STRUCTURE_PRIORITY (结构优先级)
    └── 定义产品结构的优先级

T_CX_TREAD_PARKING_CONFIG (胎面停放时间配置)
    └── 定义胎面停放时间约束
```

### 4.3 主数据关系

```
T_MDM_MOLDING_MACHINE (成型机台)
    ├── 1:1 → T_MDM_CX_MACHINE_FIXED (成型固定机台)
    └── N:1 → T_MDM_BOM_INFO (BOM信息)

T_MDM_MATERIAL_INFO (物料信息)
    ├── 1:1 → T_MDM_SKU_STRUCTURE_REF (SKU与结构关系)
    ├── 1:1 → T_MDM_SKU_CONSTRUCTION_REF (SKU与施工关系)
    └── N:1 → T_MDM_BOM_INFO (BOM信息)

T_MDM_WORK_CALENDAR (工作日历)
    └── 定义各工序的工作日历

T_MDM_STRUCTURE_NAME (结构名称字典)
    └── N:1 → T_MDM_STRUCTURE_LH_RATIO (成型结构硫化配比)
```

## 八、性能优化建议

### 8.1 索引优化

1. **高频查询字段**：SCHEDULE_DATE、CX_MACHINE_CODE、EMBRYO_CODE 建立索引
2. **复合索引**：对于经常一起查询的字段建立复合索引
3. **唯一索引**：MATERIAL_CODE、MACHINE_CODE 等唯一标识字段

### 8.2 分区策略

1. **按日期分区**：T_CX_SCHEDULE_RESULT 和 T_CX_SCHEDULE_DETAIL 按月分区
2. **按机台分区**：如果数据量很大，可考虑按机台编号进行子分区

### 8.3 查询优化

1. **避免全表扫描**：使用索引字段作为查询条件
2. **批量操作**：使用批量插入和批量更新减少数据库交互
3. **读写分离**：考虑将报表查询分流到从库
