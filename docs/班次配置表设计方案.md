# 班次配置表设计方案

## 一、需求说明

### 1.1 业务场景
- 连续排程3天，共8个班次
- 第一天排2个班（早班、中班），无夜班
- 第二天、第三天各排3个班（夜、早、中）
- 不同工厂配置可能不同（一天2班或3班）
- 班次时间可配置

### 1.2 班次映射关系

| 班次号 | 排程天数 | 班次类型 | 时间范围 | 跨天 |
|-------|---------|---------|---------|-----|
| CLASS1 | 第1天 | 早班(DAY) | 06:00-14:00 | 否 |
| CLASS2 | 第1天 | 中班(AFTERNOON) | 14:00-22:00 | 否 |
| CLASS3 | 第2天 | 夜班(NIGHT) | 22:00-06:00 | 是 |
| CLASS4 | 第2天 | 早班(DAY) | 06:00-14:00 | 否 |
| CLASS5 | 第2天 | 中班(AFTERNOON) | 14:00-22:00 | 否 |
| CLASS6 | 第3天 | 夜班(NIGHT) | 22:00-06:00 | 是 |
| CLASS7 | 第3天 | 早班(DAY) | 06:00-14:00 | 否 |
| CLASS8 | 第3天 | 中班(AFTERNOON) | 14:00-22:00 | 否 |

### 1.3 排程逻辑说明

```
排产从早班开始：
- 第1天：早班 → 中班（跳过夜班，因为排产从早班开始）
- 第2天：夜班 → 早班 → 中班
- 第3天：夜班 → 早班 → 中班
```

## 二、表结构设计

### 方案A：扩展现有表（推荐）

在 `T_CX_SHIFT_CONFIG` 表基础上增加字段：

```sql
-- 班次配置表（扩展版）
CREATE TABLE T_CX_SHIFT_CONFIG (
    ID              BIGINT(20)      NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    
    -- 基础班次信息
    SHIFT_CODE      VARCHAR(20)     NOT NULL COMMENT '班次编码：NIGHT/DAY/AFTERNOON',
    SHIFT_NAME      VARCHAR(50)     COMMENT '班次名称：夜班/早班/中班',
    SHIFT_ORDER     INT(11)         COMMENT '班次序号（单日内排序）：1-夜班 2-早班 3-中班',
    
    -- 时间配置
    START_TIME      VARCHAR(8)      COMMENT '开始时间（HH:mm:ss格式）',
    END_TIME        VARCHAR(8)      COMMENT '结束时间（HH:mm:ss格式）',
    SHIFT_HOURS     INT(11)         COMMENT '班次时长（小时）',
    IS_CROSS_DAY    INT(1)          DEFAULT 0 COMMENT '是否跨天：0-否 1-是',
    
    -- 排程周期配置（新增）
    SCHEDULE_DAY    INT(11)         COMMENT '排程天数：1-第一天 2-第二天 3-第三天',
    DAY_SHIFT_ORDER INT(11)         COMMENT '当天班次序号：第几个班',
    CLASS_FIELD     VARCHAR(20)     COMMENT '对应结果表字段：CLASS1~CLASS8',
    
    -- 多工厂支持
    FACTORY_CODE    VARCHAR(20)     COMMENT '工厂编号',
    
    -- 启用状态
    IS_ACTIVE       INT(1)          DEFAULT 1 COMMENT '是否启用：0-禁用 1-启用',
    
    -- 审计字段
    REMARK          VARCHAR(500)    COMMENT '备注',
    CREATE_BY       VARCHAR(50)     COMMENT '创建人',
    CREATE_TIME     DATETIME        DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    UPDATE_BY       VARCHAR(50)     COMMENT '更新人',
    UPDATE_TIME     DATETIME        DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    PRIMARY KEY (ID),
    INDEX IDX_FACTORY_DAY (FACTORY_CODE, SCHEDULE_DAY),
    INDEX IDX_CLASS_FIELD (CLASS_FIELD)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='班次配置表';
```

## 四、排程算法改造

### 4.1 加载班次配置

```java
/**
 * 加载班次配置（按工厂）
 * @param factoryCode 工厂编号
 * @return 排序后的班次配置列表
 */
public List<CxShiftConfig> loadShiftConfigs(String factoryCode) {
    return shiftConfigMapper.selectList(
        new LambdaQueryWrapper<CxShiftConfig>()
            .eq(CxShiftConfig::getFactoryCode, factoryCode)
            .eq(CxShiftConfig::getIsActive, 1)
            .orderByAsc(CxShiftConfig::getScheduleDay)
            .orderByAsc(CxShiftConfig::getDayShiftOrder)
    );
}
```

### 4.2 按天执行排程

```java
/**
 * 执行排程（连续3天）
 */
public List<CxScheduleResult> executeSchedule(ScheduleContextDTO context) {
    List<CxScheduleResult> allResults = new ArrayList<>();
    
    // 加载班次配置
    List<CxShiftConfig> shiftConfigs = loadShiftConfigs(context.getFactoryCode());
    
    // 按天数分组
    Map<Integer, List<CxShiftConfig>> dayShiftMap = shiftConfigs.stream()
        .collect(Collectors.groupingBy(CxShiftConfig::getScheduleDay));
    
    // 连续执行3天排程
    for (int day = 1; day <= 3; day++) {
        List<CxShiftConfig> dayShifts = dayShiftMap.get(day);
        if (CollectionUtils.isEmpty(dayShifts)) {
            continue;
        }
        
        // 设置当前天的排程日期
        LocalDate currentScheduleDate = context.getScheduleDate().plusDays(day - 1);
        context.setCurrentScheduleDay(day);
        context.setCurrentScheduleDate(currentScheduleDate);
        context.setCurrentShiftConfigs(dayShifts);
        
        // 执行该天的排程
        List<CxScheduleResult> dayResults = executeDaySchedule(context, day, dayShifts);
        allResults.addAll(dayResults);
    }
    
    return allResults;
}

/**
 * 执行单天排程
 */
private List<CxScheduleResult> executeDaySchedule(ScheduleContextDTO context, int day, List<CxShiftConfig> dayShifts) {
    // 第一步：计算日胎胚任务
    List<DailyEmbryoTask> tasks = calculateDailyEmbryoTasks(context);
    
    // 第二步：分配任务到机台
    List<MachineAllocationResult> allocations = allocateTasksToMachines(tasks, context);
    
    // 第三步：班次均衡分配（按该天实际班次配置）
    List<ShiftAllocationResult> shiftAllocations = balanceShiftAllocation(allocations, dayShifts, context);
    
    // 第四步：排生产顺位
    List<CxScheduleDetail> details = calculateSequence(shiftAllocations, context);
    
    // 构建结果（映射到CLASS1~CLASS8）
    return buildScheduleResults(context, allocations, shiftAllocations, details, dayShifts);
}
```

### 4.3 班次映射

```java
/**
 * 将班次分配结果映射到结果表字段
 */
private void mapToClassFields(CxScheduleResult result, ShiftAllocationResult shiftAllocation, List<CxShiftConfig> dayShifts) {
    // 获取该天的所有班次
    for (CxShiftConfig shiftConfig : dayShifts) {
        String classField = shiftConfig.getClassField();  // 如 "CLASS1"
        Integer shiftQty = shiftAllocation.getShiftPlanQty().get(shiftConfig.getShiftCode());
        
        // 通过反射或switch设置对应字段
        setClassFieldValue(result, classField, shiftQty);
    }
}

private void setClassFieldValue(CxScheduleResult result, String classField, Integer qty) {
    switch (classField) {
        case "CLASS1": result.setClass1PlanQty(BigDecimal.valueOf(qty)); break;
        case "CLASS2": result.setClass2PlanQty(BigDecimal.valueOf(qty)); break;
        case "CLASS3": result.setClass3PlanQty(BigDecimal.valueOf(qty)); break;
        case "CLASS4": result.setClass4PlanQty(BigDecimal.valueOf(qty)); break;
        case "CLASS5": result.setClass5PlanQty(BigDecimal.valueOf(qty)); break;
        case "CLASS6": result.setClass6PlanQty(BigDecimal.valueOf(qty)); break;
        case "CLASS7": result.setClass7PlanQty(BigDecimal.valueOf(qty)); break;
        case "CLASS8": result.setClass8PlanQty(BigDecimal.valueOf(qty)); break;
    }
}
```

## 六、总结

### 需要新增的字段：
1. `SCHEDULE_DAY` - 排程天数（第几天）
2. `DAY_SHIFT_ORDER` - 当天班次序号
3. `CLASS_FIELD` - 对应结果表字段
4. `FACTORY_CODE` - 工厂编号

### 需要修改的代码：
1. `CxShiftConfig` 实体类 - 新增4个字段
2. `CoreScheduleAlgorithmServiceImpl.executeSchedule` - 改为循环3天执行
3. `balanceShiftAllocation` - 接收动态班次配置
4. `buildScheduleResults` - 按CLASS_FIELD映射结果

### 数据库变更：
```sql
ALTER TABLE T_CX_SHIFT_CONFIG 
ADD COLUMN SCHEDULE_DAY INT(11) COMMENT '排程天数：1-第一天 2-第二天 3-第三天' AFTER IS_CROSS_DAY,
ADD COLUMN DAY_SHIFT_ORDER INT(11) COMMENT '当天班次序号' AFTER SCHEDULE_DAY,
ADD COLUMN CLASS_FIELD VARCHAR(20) COMMENT '对应结果表字段：CLASS1~CLASS8' AFTER DAY_SHIFT_ORDER,
ADD COLUMN FACTORY_CODE VARCHAR(20) COMMENT '工厂编号' AFTER CLASS_FIELD;

-- 添加索引
CREATE INDEX IDX_FACTORY_DAY ON T_CX_SHIFT_CONFIG(FACTORY_CODE, SCHEDULE_DAY);
CREATE INDEX IDX_CLASS_FIELD ON T_CX_SHIFT_CONFIG(CLASS_FIELD);
```
