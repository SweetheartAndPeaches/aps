## 二、核心算法一致性核对

### 2.1 试错分配算法核对

#### 文档要求（4.2节）

```
算法核心思想：试错+择优
1. 递归回溯尝试所有可能的分配方案
2. 约束检查：机台状态、SKU种类限制、结构切换限制、产能限制
3. 最优方案判断：负载差最小、种类差最小
4. 回溯机制：保存快照、失败回溯
```

#### 实际实现（AlgorithmService.java）

| 文档要求 | 实现代码 | 一致性 |
|----------|----------|--------|
| 递归回溯 | `recursiveAllocate()` 方法 | ✅ 一致 |
| 约束检查-机台状态 | `canAllocateToMachine()` 第1步 | ✅ 一致 |
| 约束检查-SKU种类 | `MAX_SKU_PER_MACHINE_PER_DAY = 4` | ✅ 一致 |
| 约束检查-结构切换 | `MAX_STRUCTURE_SWITCH_PER_DAY = 2` | ✅ 一致 |
| 约束检查-产能限制 | `machine.getMaxDailyCapacity()` | ✅ 一致 |
| 回溯机制-保存快照 | `saveSnapshot()` 方法 | ✅ 一致 |
| 回溯机制-恢复快照 | `restoreSnapshot()` 方法 | ✅ 一致 |
| 负载均衡优先 | 按负载升序排序候选机台 | ✅ 一致 |

**核对结论**: ✅ 试错分配算法与文档要求**完全一致**

### 2.2 班次均衡调整算法核对

#### 文档要求（4.3节）

```
核心逻辑：
1. 计算各班次产量和偏差
2. 偏差分级处理：
   - < 5%: 正常范围，不调整
   - 5%-20%: 中度偏差，可选调整
   - > 20%: 严重偏差，强制调整
3. 波浪交替分配（总整车数 > 11时）：
   - 早班 = 6车
   - 中班 = 总数 - 12车
   - 夜班 = 6车
```

#### 实际实现（AlgorithmService.java）

| 文档要求 | 实现代码 | 一致性 |
|----------|----------|--------|
| 计算班次偏差 | `calculateDeviations()` 方法 | ✅ 一致 |
| 偏差阈值5% | `totalQuantity * 0.05` | ✅ 一致 |
| 解析班次比例 | `parseShiftRatio()` 方法 | ✅ 一致 |
| 计算目标分布 | `calculateTargetDistribution()` 方法 | ✅ 一致 |
| 执行调整 | `performShiftAdjustment()` 方法 | ✅ 一致 |
| 波浪交替分配 | 比例参数 "1:2:1" 支持 | ⚠️ 部分实现 |

**差异说明**:
- 文档要求波浪交替分配的阈值判断（整车数 > 11）和具体分配逻辑（早班6车、夜班6车）
- 当前实现使用配置的比例参数（默认"1:2:1"），但未实现阈值判断逻辑

**核对结论**: ⚠️ 班次均衡调整算法**基本一致**，波浪交替分配细节需补充

4. **实现观察者模式**
